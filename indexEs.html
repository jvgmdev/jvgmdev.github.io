Proyectos

Caso de Estudio

Data Visibility Engine DVE

Framework de seguridad a nivel de registro configurable para ServiceNow

Problema

Las instancias empresariales necesitan control granular sobre qué registros puede ver cada usuario. Las ACLs nativas son rígidas y difíciles de mantener a escala - no había una forma reutilizable y amigable para el administrador de aplicar visibilidad a nivel de registro sin escribir código personalizado por caso de uso.

Solución

Se construyó un framework configurable de Before Query Business Rules que aplica la visibilidad de registros en tiempo de consulta. Soporta múltiples tipos de coincidencia incluyendo User Criteria, utiliza caché de sesión del lado del servidor para el rendimiento, e incluye una interfaz de administración para gestionar reglas sin scripting, además de una herramienta de simulación para probar la lógica de visibilidad antes del despliegue.

Before Query Business Rules Session Cache GlideRecord User Criteria JavaScript Script Includes Admin UI
